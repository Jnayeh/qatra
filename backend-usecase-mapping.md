## Critical Gaps (Highest Priority)

### 1. GDPR 30-Day Auto-Anonymization (SA06 partial, D11 partial)
- GDPRDeletionRequest is created but never automatically processed
- No `@Scheduled` task scans for 30-day-old IN_PROGRESS requests to anonymize PII
- **Fix needed:** Add a `GDPRProcessingScheduler` that runs daily, finds requests older than 30 days, anonymizes User + DonorProfile PII, and marks request as COMPLETED

### 2. Quiet Hours & Frequency Preferences Not Enforced (D08 partial, D20 partial, SYS06 partial)
- DonorProfile stores `NotificationPreferences` with `QuietHours` and `NotificationFrequency`
- notification-service does NOT check these before dispatching
- **Fix needed:** NotificationDispatchService should query donor preferences and suppress/honor quiet hours and frequency settings

### 3. Server-Side Session Management (SYS01 partial)
- Class diagram defines Session entity with accessTokenHash, refreshTokenHash, expiresAt
- Implementation is pure stateless JWT — no server-side session tracking
- Cannot revoke individual sessions or enforce server-side expiry
- **Fix needed:** Implement SessionService with JPA persistence, session cleanup scheduler, and per-session revocation

### 4. System Dashboard & Platform Reports (SA10, SA11, SA13)
- SA10 (system dashboard): `GET /analytics/dashboard` ✅ Implemented
- SA11 (system health): ❌ Removed from backend scope
- SA13 (platform reports): ❌ Removed from backend scope

### 5. MetricsSnapshot & DemandForecast (SYS15, SYS16)
- SYS15 (batch analytics): Still uses `@Scheduled` cron jobs for real-time metrics aggregation — not a direct API endpoint
- SYS16 (demand forecasting): ❌ Removed from backend scope

### 6. Staff Activity Log for Center Admins (CA07)
- CA07 (staff activity log): `GET /analytics/audit-logs?centerId=X` ✅ Implemented — joins with `user_roles` table to filter by center
- SA12 (export audit logs): `GET /analytics/audit-logs/export` ✅ Implemented — returns CSV with centerId filter support

### 7. Staff View Donor Health Questionnaire (CS08 partial)
- Staff can view donor profile and eligibility but cannot view the full HealthQuestionnaire
- **Fix needed:** Add endpoint like `GET /donors/{id}/health-questionnaire` with CENTER_STAFF authorization

### 8. Profile Completion Nudge Reminders (D30 partial)
- Appointment and eligibility reminders exist
- No profile completion nudge for donors with incomplete profiles
- **Fix needed:** Add a scheduler that identifies donors with `profileComplete=false` and sends profile nudge notifications
